"Mastering Ruby Closures"を読んだ

Dec 24th, 2017 1:19 pm | Comments

“Mastering Ruby Closures"を読んだ。

Mastering Ruby Closures: A Guide to Blocks, Procs, and Lambdas

クロージャの観点からRubyのBlock、Proc、Lambdaを解説する本である。

1章でクロージャの基本を押さえ、2章でBlock、3章でProcとLambdaが解説されている。4章は、3章までの知識を使って自前のLazy Enumerableを実装する。

わかりそうでわからないクロージャの定義が簡潔に説明されている。本文中でのクロージャの定義は以下の通り。

  1. It needs to be a function…
  2. whose body references some variable that…
  3. is declared in a parent scope.

もうひとつスッキリしない言葉として「レキシカルスコープ」があったが、これも一文で説明されている。

lexical scoping says that whichever assignment to a variable that is the closest gets that value.

残念ながら、こちらはあまりピンと来なかった。なんとなくは分かるのだけれども。

100ページ程度のさっと読める本であるけれど内容は濃いと思う。4章をちゃんと理解したいので、もう一度読み返すつもりである。

Practical Monitoring: Effective Strategies for the Real World

"Implementing DevOps With Ansible 2"を読んだ

Nov 11th, 2017 5:33 am | Comments

“Implementing DevOps with Ansible 2"を読んだ。

Implementing DevOps with Ansible 2: A step-by-step guide to automating all DevOps stages with ease using Ansible

“Ansible Up and Running"のついでに読んでみた。タイトルの通り、DepOpsの文脈の中でAnsibleをどのように使うかという観点の本である。

CI/CDのパイプラインにおけるAnsibleの役割として、以下の6つの例が挙げられている。

  1. テスト環境のプロヴィジョニング
  2. デプロイ環境のプロヴィジョニング
  3. デプロイの実施
  4. デプロイ後のアプリケーションのローンチ
  5. 障害発生時の環境のロールバック
  6. プロダクション環境へのインクリメンタルなリリース（rollout）

上記の他にAnsibleとVagrantを連携してローカルの開発環境を構築する例も紹介されている。

また、Ansibleの運用に関するベストプラクティスとして以下の9つが挙げられている。

  1. Ansibleのplaybookは常にバージョン管理する
  2. Ansibleのplaybookを製品（artifacts）に同梱する
  3. 環境ごとにinventoryファイルを準備する
  4. 開発環境とプロダクション環境のデプロイに可能な限り同じplaybookを使う
  5. インフラ周りの同期にAnsibleを活用する
  6. playbookを極力シンプルに保つ
  7. 再利用可能な自動化の実現のためにroleを利用する
  8. 環境（開発、QA、プロダクション）を常に同期させる

これくらい活用できたら楽しいだろうな。

"The Way of the Web Tester"を読んだ

Oct 27th, 2017 3:58 am | Comments

“The Way of The Web Tester”を読んだ。

The Way of the Web Tester: A Beginner’s Guide to Automating Tests

「初めての自動テスト」というタイトルで日本語訳が出版されている。

初めての自動テスト ―Webシステムのための自動テスト基礎

ウェブシステムを対象とした自動テストの入門書である。

前半では、自動テストをUIテスト、インテグレーションテスト、ユニットテスト（テストピラミッドと呼ぶ）に分類し、それぞれのテストの概要と実践方法が解説されている。

また、テストピラミッドの組み合わせ方について説明されている。テスト項目はできるだけ下のレイヤに落とし込み、上のレイヤのテストを作りすぎない、など。

後半では、テストコードのプログラミングテクニックが解説されており、最終章ではテスト駆動開発が紹介されている。

自動テスト？当たり前でしょう。という人にとっては、本当に当たり前のことをまとめただけの本であると思う。一方、これまでテストコードがないレガシーなシステムを抱えていて、これから自動テストに取り組もうという現場の人にとっては、貴重な情報源になると思う。

とくに、これまで自動テストに理解がなかった人たちに、正しい形の自動テストを布教する教材として有用になることを、個人的には期待している。

「ネスペ 27 礎」、「ネスペ 26 道」、「ネスペの剣 25」を読んだ。

ネスペ 27 礎 -ネットワークスペシャリストの最も詳しい過去問解説 (情報処理技術者試験)

ネスペ 26　道　－ネットワークスペシャリストの最も詳しい過去問解説 (情報処理技術者試験)

ネスぺの剣25 ~ネットワークスペシャリストの最も詳しい過去問解説 (情報処理技術者試験)

各年度のネットワークスペシャリスト試験の午後問題を、まるごと一冊使ってみっちり解説する本である。

サブタイトルに「最も詳しい」とある通り、本当に詳しく、懇切丁寧に解説してくれる。このネスペシリーズは基本的には過去問の解説本ではあるけれど、単なる試験対策に留まらずにネットワーク知識の応用の面で良い勉強になったと思う。

試験勉強の進捗としては、午後１までは現時点で何とかなりそうな気がしている。午後２を確実に取るにはもう一歩レベルアップが必要。9月中にその辺りを対策したい。

「ネスペの基礎力」を読んだ

Sep 2nd, 2017 12:01 am | Comments

「ネスペの基礎力」を読んだ。

ネスペの基礎力 -プラス20点の午後対策 (情報処理技術者試験)

秋の情報処理技術者試験でネットワークスペシャリストの取得を狙っている。

過去問を解いてみたところ、午前問題はあまり問題ないけれど、午後問題になるとわかっているつもりでも回答が書けないことが多くあった。

基礎力が足りていないのだと思う。ということで、そのものズバリの「ネスペの基礎力」というこの本を読んでみた。

レイヤ別の基礎技術とセキュリティ、その他の関連技術（負荷分散など）が概要レベルで解説されている。

知らない話題も結構あった。知っている話題についても、改めてそれぞれのネットワーク技術の中での位置付けを確認することができた。

資格試験の醍醐味は、試験勉強を通して特定分野の知識を体系的に整理できる点だと思う。その点でこういう本が存在することはとてもありがたい。
